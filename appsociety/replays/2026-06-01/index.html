Acte 1 — C'est quoi l'attribution mobile ?

Définition : "savoir quelle source a ramené l'utilisateur, et permettre aux plateformes de trouver d'autres gens du même profil." Les 2 systèmes qui coexistent en 2026 :

Realtime / déterministeSKAdNetwork (SCAN)
L'utilisateur a accepté l'ATT (~25 %)Refus de l'ATT (~75 %)
On sait qui a installé via quelle pubOn sait combien sur un campaign_id, agrégé
Instantané, optimisation finePostback unique, fenêtre 24–72h

Acte 2 — Pourquoi c'est cassé

iOS 14.5 (2021) : Apple coupe l'IDFA, oblige à demander la permission via l'ATT prompt. ~75 % refusent. Résultat : les SDK ads ne savent plus suivre, sauf via SCAN qui agrège et masque les identités.

Le symptôme que tout le monde reconnaît : ton dashboard TikTok dit X conversions, ton Stripe en voit Y, l'écart est de 30 à 200 %. Anna/Florent sont en plein dedans le matin même — "Cinq heures du mat, lancé la campagne iOS, et là on est à 0 conversions, alors qu'il y a eu plein de téléchargements." Diagnostic Harry : SCAN met du temps, c'est normal, il faut attendre 24-72h.

Le 3e circuit, bricolé : l'attribution probabiliste des MMP (IP, modèle, opérateur) pour rattraper certains refus d'ATT. Déjà activé chez Harry via Tenjin.

Acte 3 — Comment setup

Deux chemins :

Sans MMP : tu installes le SDK propriétaire de la plateforme (TikTok, Meta…), tu testes via le QR code dans l'Event Manager, tu lances. "Gratuit, ça marche dans 60 % des cas. Et dans les 40 % restants, c'est la jungle. Personne n'en parle sur Reddit, c'est tabou comme Voldemort."

Avec MMP (le reveal) : un seul SDK qui unifie tout. Marché : Adjust, AppsFlyer (250€-1000€/mois), Singular, Branch, et le moins cher : Tenjin. Plan gratuit jusqu'à 2000 conversions/mois. Au-delà : forfait ~200€/mois pour 10K conversions = 4 centimes par event vs 7+ chez les autres.

Setup en 4 étapes côté Tenjin :

  1. SDK install dans l'app, events critiques firés (install, trial_started, purchase)
  2. Vérif dans Diagnose → Live Event Tool que les events arrivent
  3. Section callback de Tenjin : on mappe ses events custom vers les events natifs des plateformes (trial_started → start_trial TikTok)
  4. Côté TikTok Event Manager : on connecte le MMP via clés API, on choisit le partenaire de mesure

Choix de l'event à optimiser : grand débat live. Valentin tranche pour Livio : "Fais l'install au début, pas le start_trial. Tu auras des résultats moins bons, mais tu en auras. Si tu pars sur start_trial sans volume, tu auras rien, tu sauras pas si tu fais bien." Harry confirme : il est lui-même passé install → trial APRÈS avoir eu du volume, ce qui a augmenté le CPI mais amélioré le trial-to-paid.

Échanges-clés du Q&A

  • Livio + Capacitor : Valentin enquête live, pas de SDK TikTok officiel pour Capacitor. Mais Tenjin a un plugin Capacitor à jour (dernier release 4 jours avant le call). Livio décide : il part sur Tenjin direct.
  • Snapchat ? Question d'Elyass (qui pense à tester après la stat 91 % des 15-24 ans entendue au 18/05). Réponse : Snapchat a peut-être un SDK, mais Tenjin gère Snapchat aussi. L'avantage du MMP, c'est justement de pouvoir diversifier les plateformes sans refaire l'attribution à chaque fois.
  • Sylvain : "C'est vraiment nécessaire dès le départ ?" — non, tu peux te baser uniquement sur SCAN, qui marche par défaut. Mais en dessous d'un seuil de volume, SCAN devient muet (protection vie privée Apple).
